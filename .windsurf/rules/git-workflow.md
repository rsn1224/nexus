# Git 作業ルール

## 実装完了後のコミット手順

### ✅ 正しい手順

```bash
# 1. 変更内容を確認しながらステージング
git add -p

# 2. コミット
git commit -m "feat: xxx"

# 3. push
git push
```

### ❌ 禁止

```bash
git add -A   # 全ファイル一挡ステージング禁止
git add .    # 同上
```

`git add -A` / `git add .` を使うと不要なファイル（`*_old.ts` ・残ったデバッグファイル等）がコミットに混入するため禁止。
**必ず `git add -p` で変更をブロック単位に確認してからステージングすること。**

## ファイル删除の原則

- `*_old.*` / `*_backup.*` / `*.bak` のファイルは作成禁止
- リファクタリング中に旧ファイルを残す場合は必ず削除してからコミットする

## コミットメッセージ規約

```
feat: 新機能追加
fix:  バグ修正
refactor: リファクタリング
chore: 設定・ツール変更
docs: ドキュメント修正
test: テスト追加・修正
```
