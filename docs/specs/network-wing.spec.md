# Network Wing（ネットワーク設定） — 仕様書

## 概要
DNS 設定・Ping 測定と TCP チューニングの 2 タブで構成されるネットワーク最適化画面。ゲーミング向けの低レイテンシ設定をワンクリックで適用できる。

## 前提条件
- 管理者権限でネットワーク設定を変更可能であること
- ネットワークアダプター情報が取得可能であること
- DNS 設定、TCP パラメータの読み書きがバックエンドで実装されていること
- Ping コマンドが実行可能であること

## シナリオ

### DNS & Ping タブ

#### Scenario 1: アダプター情報の表示
- **Given** DNS & Ping タブが選択されている
- **When** ADAPTER セクションがレンダリングされる
- **Then** プライマリアダプター名、IP アドレス、MAC アドレス、ステータスが表示される

#### Scenario 2: 現在の DNS 表示
- **Given** DNS セクションが表示中
- **When** 現在の DNS 設定を取得する
- **Then** 現在のプライマリ/セカンダリ DNS アドレスが表示される

#### Scenario 3: DNS プリセットの適用
- **Given** DNS セクションのプリセットドロップダウンが表示中
- **When** Cloudflare プリセットを選択して適用する
- **Then** DNS が 1.1.1.1 / 1.0.0.1 に設定される

#### Scenario 4: DNS プリセット一覧
- **Given** プリセットドロップダウンを開く
- **When** 選択肢を確認する
- **Then** Cloudflare、Google、OpenDNS、Quad9 が選択可能

#### Scenario 5: カスタム DNS の設定
- **Given** DNS セクションでカスタム入力が表示中
- **When** プライマリ・セカンダリの DNS アドレスを手動入力して適用する
- **Then** 入力した DNS アドレスが設定される

#### Scenario 6: Ping 測定
- **Given** PING セクションが表示中（デフォルトターゲット: 8.8.8.8）
- **When** Ping を実行する
- **Then** レイテンシ結果がカラーインジケーター付きで表示される

#### Scenario 7: Ping ターゲットの変更
- **Given** PING セクションのターゲット入力が表示中
- **When** ターゲットアドレスを変更して Ping を実行する
- **Then** 指定アドレスへの Ping 結果が表示される

### TCP チューニングタブ

#### Scenario 8: DNS キャッシュのフラッシュ
- **Given** TCP チューニングタブが選択されている
- **When** DNS キャッシュフラッシュボタンを押す
- **Then** DNS キャッシュがクリアされ、結果が表示される

#### Scenario 9: ネットワーク設定の個別適用
- **Given** ネットワーク設定リストが表示中
- **When** 特定の設定の適用ボタンを押す
- **Then** その設定のみが適用される

#### Scenario 10: ネットワーク設定の個別リバート
- **Given** ネットワーク設定が変更済み
- **When** 特定の設定のリバートボタンを押す
- **Then** その設定のみがデフォルト値に戻される

#### Scenario 11: Nagle アルゴリズムのトグル
- **Given** TCP チューニング設定が表示中
- **When** Nagle アルゴリズムを無効化する
- **Then** TCP_NODELAY が有効化され、低レイテンシモードになる

#### Scenario 12: Delayed ACK のトグル
- **Given** TCP チューニング設定が表示中
- **When** Delayed ACK を無効化する
- **Then** ACK 遅延が解除される

#### Scenario 13: ネットワークスロットリングプリセット
- **Given** スロットリングプリセットが表示中
- **When** プリセットを選択する
- **Then** 対応するスロットリング設定が適用される

#### Scenario 14: QoS 帯域幅の設定
- **Given** QoS 設定が表示中
- **When** 帯域幅 % を入力する
- **Then** QoS 帯域幅制限が設定される

#### Scenario 15: TCP 自動チューニングレベルの設定
- **Given** TCP 自動チューニング設定が表示中
- **When** チューニングレベルを選択する
- **Then** TCP 自動チューニングが指定レベルに設定される

#### Scenario 16: ゲーミングプリセットのワンクリック適用
- **Given** TCP チューニングタブが表示中
- **When** ゲーミングプリセットボタンを押す
- **Then** Nagle 無効、Delayed ACK 無効、最適スロットリング等がまとめて適用される

#### Scenario 17: デフォルトへのリセット
- **Given** TCP 設定が変更済み
- **When** リセットボタンを押す
- **Then** すべてのネットワーク設定がデフォルト値に戻される

#### Scenario 18: ネットワーク品質の測定
- **Given** TCP チューニングタブが表示中
- **When** ネットワーク品質測定を実行する
- **Then** レイテンシとジッターが測定・表示される

## エッジケース
- 管理者権限がない場合、設定変更ボタンを無効化しツールチップで理由を表示する
- ネットワークアダプターが見つからない場合、エラーメッセージを表示する
- 無効な IP アドレスが DNS に入力された場合、バリデーションエラーを表示する
- Ping ターゲットが到達不能な場合、タイムアウトメッセージを表示する
- VPN 使用中の場合、アダプター情報が VPN のものになることを注意表示する
- ゲーミングプリセット適用中にエラーが発生した場合、適用済み設定をロールバックする

## 非機能要件
- DNS 変更は適用後 5 秒以内に反映を確認する
- Ping 測定のタイムアウトは 5 秒
- ネットワーク設定の変更前にバックアップを取得すること
- TCP パラメータ変更はレジストリ操作を伴うため、変更前の値を保存する
- ゲーミングプリセットの適用はアトミックに行う（途中失敗時はロールバック）
