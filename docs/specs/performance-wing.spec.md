# Performance Wing（パフォーマンス） — 仕様書

## 概要
プロセス管理、ゲームプロファイル、ウォッチドッグ、セッション履歴の 4 タブで構成されるパフォーマンス最適化画面。プロセスの優先度変更・終了、ゲーム別最適化プロファイルの管理、自動監視ルール、過去セッションの比較分析を提供する。

## 前提条件
- バックエンドがプロセス一覧を取得・操作できること（優先度変更、終了）
- プロファイルデータ（`profiles.json`）が読み書き可能であること
- `nexus://game-launched` / `nexus://game-exited` イベントが発行されること
- セッション履歴データが永続化されていること

## シナリオ

### Process タブ

#### Scenario 1: プロセステーブルの表示とソート
- **Given** Process タブが選択されている
- **When** テーブルヘッダー（NAME / CPU% / MEM / DISK）をクリックする
- **Then** 対応するカラムで昇順・降順にソートされる

#### Scenario 2: プロセス名フィルター
- **Given** プロセステーブルが表示中
- **When** フィルター入力にテキストを入力する
- **Then** プロセス名が部分一致するプロセスのみ表示される

#### Scenario 3: CPU 閾値スライダー
- **Given** CPU 閾値スライダーがデフォルト 15% に設定
- **When** スライダーを 30% に変更する
- **Then** CPU 使用率 30% 以上のプロセスのみ表示される

#### Scenario 4: プロセス行展開と優先度変更
- **Given** プロセス行が折りたたまれている
- **When** 行を展開し HIGH ボタンを押す
- **Then** 対象プロセスの優先度が HIGH に変更され、結果が反映される

#### Scenario 5: プロセス終了（KILL）
- **Given** プロセス行が展開されている
- **When** KILL ボタンを押す
- **Then** 対象プロセスが終了され、テーブルから削除される

#### Scenario 6: RUN BOOST 実行
- **Given** Process タブが表示中
- **When** RUN BOOST ボタンを押す
- **Then** 最適化が実行され、BoostResultsPanel に結果が表示される

### Profiles タブ

#### Scenario 7: プロファイル一覧の表示
- **Given** Profiles タブが選択されている
- **When** 保存済みプロファイルが存在する
- **Then** GameProfile のリストが表示され、各項目に適用・編集・削除ボタンがある

#### Scenario 8: プロファイルの作成
- **Given** ProfileForm が開かれている
- **When** ゲーム名、EXE パス、CPU アフィニティ、プロセス優先度、電源プラン、サスペンド/キルリスト、タイマー解像度、ブーストレベル、コアパーキングを入力して保存する
- **Then** 新しいプロファイルがリストに追加される

#### Scenario 9: AffinityPanel でコア選択
- **Given** ProfileForm の AffinityPanel が表示中
- **When** CoreSelector で特定のコアを選択/解除する
- **Then** CPU アフィニティマスクが更新される

#### Scenario 10: プロファイルの適用
- **Given** プロファイルリストが表示中
- **When** プロファイルの適用ボタンを押す
- **Then** プロファイルの設定（優先度、アフィニティ等）が対象プロセスに反映される

#### Scenario 11: プロファイルのインポート/エクスポート
- **Given** ProfileSharePanel が表示中
- **When** エクスポートを実行する
- **Then** プロファイルが JSON 形式でエクスポートされる

#### Scenario 12: ゲーム起動/終了イベントへの反応
- **Given** Profiles タブがアクティブ
- **When** `nexus://game-launched` イベントを受信する
- **Then** 該当ゲームのプロファイルが自動適用される（存在する場合）

### Watchdog タブ

#### Scenario 13: ウォッチドッグルール一覧
- **Given** Watchdog タブが選択されている
- **When** ルールが登録済み
- **Then** WatchdogRuleTable にルール一覧がトグル・編集・削除ボタン付きで表示される

#### Scenario 14: ウォッチドッグルールの作成
- **Given** WatchdogRuleModal が開かれている
- **When** ルール名、条件、アクション、プロセスフィルター、クールダウン時間を入力して保存する
- **Then** 新しいルールがテーブルに追加される

#### Scenario 15: ウォッチドッグルールのトグル
- **Given** ルールが有効状態
- **When** トグルを OFF にする
- **Then** ルールが無効化され、条件を満たしてもアクションが実行されない

#### Scenario 16: ウォッチドッグイベントログ
- **Given** ウォッチドッグが稼働中
- **When** ルールの条件が満たされアクションが実行される
- **Then** WatchdogEventLog にイベントがタイムライン形式で記録される

### Session タブ

#### Scenario 17: セッション履歴の表示
- **Given** Session タブが選択されている
- **When** 過去のセッションが存在する
- **Then** SessionListView にスクロール可能な履歴一覧が表示される

#### Scenario 18: セッション比較
- **Given** 2 つのセッションが選択されている
- **When** 比較ビューを開く
- **Then** SessionCompareView にサイドバイサイドでデルタメトリクスが表示される

## エッジケース
- CPU 閾値スライダーの範囲: 1〜100%（範囲外の値は丸める）
- プロファイルの EXE パスが無効な場合、適用時にエラーを表示する
- システムプロセス（PID < 10 等）への KILL は警告を出す
- ウォッチドッグルールのクールダウン中に再トリガーされた場合、アクションをスキップする
- セッション履歴が 0 件の場合、空状態メッセージを表示する
- プロファイル JSON のインポート時、スキーマ検証に失敗したらエラーを表示する
- `nexus://game-exited` 受信時に該当プロファイルの設定を元に戻す

## 非機能要件
- プロセステーブルの更新は 2 秒以内の遅延
- プロファイルの適用は 500ms 以内に完了すること
- ウォッチドッグの条件評価は 1 秒間隔
- セッション比較のデルタ計算は UI スレッドをブロックしないこと
- プロファイル JSON のバリデーションはインポート時に必ず実行する
