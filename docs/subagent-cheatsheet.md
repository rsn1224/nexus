# Claude Code サブエージェント チートシート

## 基本コマンド

### Explore サブエージェント（コードベース探索）
読み取り専用。Haiku モデルで高速。メインコンテキストを汚さない。

用途: 依存関係調査、型定義の検索、ファイル構造の把握

### Plan サブエージェント（実装計画）
読み取り専用。メインセッションのモデルを継承。

用途: 大規模タスクの実装手順を策定

## nexus でよく使うパターン

### パターン1: バグ調査
Explore サブエージェントを使って、
src/stores/useOpsStore.ts の processes が空になる原因を調べてください。
関連するコンポーネントと Rust コマンド（ops.rs）の呼び出しフローを追跡してください。

### パターン2: リファクタ前の影響調査
Explore サブエージェントを使って、
usePulseStore を import している全ファイルを列挙し、
各ファイルでどのフィールド（snapshots, error, isLoading）を使っているか報告してください。

### パターン3: テストカバレッジ分析
Explore サブエージェントを使って、
src/stores/ 配下の全ストアファイルに対応するテストファイルが
src/__tests__/ に存在するか確認し、テストが無いストアを一覧にしてください。

### パターン4: 新 Wing 追加前の調査
Explore サブエージェントを使って、
Shell.tsx の SIDEBAR_ZONES と WingId 型の定義を確認し、
新しい Wing を追加する際に変更が必要なファイルを全て列挙してください。

### パターン5: Rust 側の依存調査
Explore サブエージェントを使って、
src-tauri/src/commands/ 配下の全 .rs ファイルで
tauri::command マクロが付いている関数を一覧にしてください。
各関数の引数と戻り値の型も報告してください。

## 注意事項
- Explore は読み取り専用。ファイル変更はできない
- 探索結果の要約だけがメインセッションに返る
- 大量のファイルを探索してもメインのコンテキストは消費しない
- 修正が必要な場合はメインセッションで実行すること
