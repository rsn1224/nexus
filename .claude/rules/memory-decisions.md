# メモリ管理ルール

## 保存すべきもの

| タイプ | 内容 | 例 |
| ------ | ---- | -- |
| `user` | ユーザーの役割・好み・スキルレベル | 「Rust は初心者、React は熟練」 |
| `feedback` | 修正された行動パターン（繰り返し防止） | 「サマリーを末尾に書かない」 |
| `project` | 進行中の意思決定・背景 | 「認証リライトはコンプライアンス要件が動機」 |
| `reference` | 外部リソースの場所 | 「バグは Linear INGEST プロジェクトで管理」 |

## 保存すべきでないもの

- コードパターン・規約・ファイルパス → コードから読める
- Git 履歴・誰が何を変えたか → `git log` が正源
- デバッグ解決策・修正レシピ → コードとコミットメッセージに記録済み
- CLAUDE.md に書いてある内容の複製
- 一時的な作業状態・現在の会話コンテキスト

## 保存のタイミング

- ユーザーが明示的に「覚えて」と言った → 即座に保存
- ユーザーが行動を修正した（「それじゃなくて...」）→ feedback として保存
- 将来の会話で役立つ設計判断が出た → project として保存
- 2回以上同じ質問をされた → user として保存（説明不足のサイン）
