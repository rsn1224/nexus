# Wrap-Up スキル

セッション終了時に呼び出す。作業をきれいにまとめて次回へバトンタッチする。

## 実行手順

### 1. 作業状態の確認

```bash
git status          # 未コミットの変更
git diff --stat     # 変更サマリー
```

### 2. HANDOFF.md の更新

- 完了したタスクのステータスを `done` に更新
- 未完了タスクに `next_steps` を記録
- 発見したバグ・注意点を `notes` セクションに追記

### 3. メモリの更新

今セッションで学んだことがあれば `C:\Users\rsn12\.claude\projects\c--Users-rsn12-dev\memory\` に保存:
- 新しいユーザーの好み → `user_*.md`
- 修正された行動パターン → `feedback_*.md`
- プロジェクトの重要な決定 → `project_*.md`

### 4. 未コミット変更の整理

```bash
git add -p    # 意図した変更のみステージング
```

コミットすべき変更がある場合、コミットメッセージ案を提示してユーザーの判断を仰ぐ。

### 5. 次回セッション向けメモ

HANDOFF.md の `next_session` セクションに記録:
- 中断した作業の再開ポイント
- 動作確認が必要な項目
- Cascade に依頼予定のタスク

## 出力形式

```
## セッションまとめ

### 完了したこと
- {タスク1}
- {タスク2}

### 未完了・次回へ
- {未完了タスク}（理由: {なぜ未完了か}）

### 注意点
- {次回セッションで気をつけること}

### git 状態
- コミット済み: {はい/いいえ}
- 未ステージ: {あれば列挙}
```
